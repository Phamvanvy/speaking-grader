☁️ TOEIC Speaking Grader — Hướng dẫn Deploy lên Cloud

Đưa FastAPI + faster-whisper + Claude API lên production một cách tối ưu (chi phí, tốc độ, độ ổn định)

0 Đặc điểm app quyết định cách deploy

Trước khi chọn nền tảng, cần hiểu 5 ràng buộc kỹ thuật của project này — chúng chi phối mọi quyết định bên dưới:

  • Model ASR nặng, nạp 1 lần/process. src/api.py nạp config + cache Whisper trong RAM/VRAM lúc khởi động → cold start chậm (vài giây ~ vài chục giây). Không hợp serverless "scale-to-zero" thuần nếu cần phản hồi tức thì.
  • CPU chạy được, GPU nhanh hơn nhiều. WHISPER_DEVICE=cpu|cuda. CPU rẻ nhưng chậm; GPU (T4/L4) tăng tốc 5–15×.
  • Whisper KHÔNG an toàn đa luồng. Đừng scale bằng nhiều thread trong 1 process — scale bằng nhiều replica, mỗi replica uvicorn --workers 1 (1 GPU/replica).
  • Request batch chạy lâu. /grade-batch chấm 40 em có thể mất nhiều phút → cần tăng timeout hoặc chuyển sang kiến trúc job + queue.
  • Phụ thuộc ffmpeg + secret + egress. faster-whisper đọc audio qua ffmpeg (phải có trong image); cần ANTHROPIC_API_KEY (secret) và mạng ra ngoài để gọi Claude.

1 Stack tối ưu — nên dùng gì

🐳 Đóng gói & Build

  • Docker (multi-stage) — môi trường giống hệt local, kèm ffmpeg.
  • Bake model Whisper vào image hoặc volume cache → tránh tải lúc cold start.
  • Registry: GHCR / Docker Hub / ECR / Artifact Registry.

🚀 Nơi chạy (Runtime)

  • CPU, dùng thưa: Cloud Run / Azure Container Apps (scale-to-zero).
  • GPU, theo lượt: Modal / RunPod / Replicate (serverless GPU).
  • Always-on đơn giản: 1 VM GPU + Docker Compose.

🔐 Secrets & Config

  • Không đưa .env thật vào image/git.
  • Dùng Secret Manager của nền tảng (GCP/AWS/Azure) hoặc biến môi trường runtime.
  • Inject ANTHROPIC_API_KEY, TOEIC_MODEL, WHISPER_* lúc deploy.

📦 Lưu trữ & Quan sát

  • Container ephemeral → đừng ghi outputs/ ra đĩa local. API đã save=False; nếu cần lưu, đẩy S3/GCS hoặc DB.
  • Log ra stdout (không chỉ logs/app.log) để cloud thu được.
  • Healthcheck: dùng sẵn GET /health làm liveness/readiness probe.

2 Kiến trúc tham chiếu

Phương án A — đồng bộ (đơn giản, hợp lớp nhỏ / chấm lẻ):

Client▶ HTTPS / LB▶ Container: uvicorn src.api:app▶ Claude API

Phương án B — bất đồng bộ (khuyến nghị cho /grade-batch lớp 40 em):

Client▶ API (nhận job, trả jobId)▶ Queue
(Redis/SQS)▶ Worker GPU
(chấm)▶ Store kết quả
(S3/DB)
Khi nào cần B? Khi request vượt timeout của nền tảng (Cloud Run tối đa 60 phút, nhiều LB/proxy chỉ 30–60 giây). Tách nhận việc khỏi xử lý: API trả ngay jobId, client poll GET /jobs/{id}; worker chấm nền và lưu kết quả.

4 Đóng gói Docker (đã có sẵn trong repo)

✅ Đã smoke-test local: server boot, GET /health & /docs trả 200, POST /grade (ASR, no_ai=true) chạy đúng. Bộ file dưới đây đã nằm sẵn trong repo: Dockerfile, .dockerignore, docker-compose.yml.

Image CPU gọn, có sẵn ffmpeg, prefetch Whisper để giảm cold start. Đổi base sang nvidia/cuda:12.x-cudnn-runtime khi chạy GPU (xem mục 6).

# ---- Dockerfile (CPU) ----
FROM python:3.11-slim

# ffmpeg cho faster-whisper đọc audio; rm cache để image nhẹ
RUN apt-get update && apt-get install -y --no-install-recommends ffmpeg \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Prefetch model Whisper vào image → không tải lúc cold start
ENV WHISPER_MODEL=base \
    WHISPER_DEVICE=cpu
RUN python -c "from faster_whisper import WhisperModel; WhisperModel('base')"

COPY src ./src

# Workers=1: Whisper không an toàn đa luồng → scale bằng replica
ENV PORT=8000
EXPOSE 8000
CMD ["sh","-c","uvicorn src.api:app --host 0.0.0.0 --port ${PORT} --workers 1"]
.dockerignore (đã có) loại: .venv/ .git/ .gitnexus/ data/ outputs/ logs/ .env __pycache__/ .pytest_cache/ docs/ tests/ *.md — giữ build context nhỏ & tránh rò rỉ key/dữ liệu vào image.
ffmpeg: ASR thực tế chạy được cả khi thiếu ffmpeg trên PATH (faster-whisper dùng PyAV đã bundle), nhưng image vẫn cài ffmpeg cho chắc với mọi định dạng/đường dẫn.
Tầng ASR fast/review KHÔNG được bundle. requirements.txt chỉ có faster-whisper; whisperx (lane review) và insanely_fast_whisper (lane fast) rất nặng + thiên GPU nên bị bỏ ra. Hệ quả: mode=fast tự fallback về default, nhưng mode=review fail cứng nếu chưa cài whisperx. Cách xử lý cho image CPU: ép cả 3 tầng về faster_whisper bằng env (docker-compose.yml đã làm sẵn) để mọi mode chạy được: 
TOEIC_ASR_BACKEND_DEFAULT=faster_whisper
TOEIC_ASR_BACKEND_FAST=faster_whisper
TOEIC_ASR_BACKEND_REVIEW=faster_whisper
Muốn dùng đúng multi-tier (fast/review riêng) thì cài thêm backend → xem mục 6.

Chạy thử local (giống production)

# Cách 1 — docker run trực tiếp
docker build -t speaking-grader .
docker run --rm -p 8000:8000 \
  -e ANTHROPIC_API_KEY=sk-ant-... \
  -e TOEIC_BACKEND=anthropic \
  -e TOEIC_MODEL=claude-sonnet-4-6 \
  speaking-grader

# Cách 2 — docker compose (tự đọc biến từ .env, có healthcheck /health)
docker compose up --build

# Kiểm tra: http://localhost:8000/health và /docs
# Không có key? gửi no_ai=true (chỉ ASR) hoặc TOEIC_BACKEND=local (llama.cpp)

5 Tối ưu chi phí & hiệu năng

⚡ Giảm cold start

  • Bake model vào image (mục 4) hoặc mount volume cache ~/.cache/huggingface.
  • Giữ min-instances ≥ 1 nếu cần phản hồi nhanh.
  • Whisper nhỏ hơn (tiny/base) nạp nhanh hơn.

💰 Giảm chi phí

  • Scale-to-zero khi rảnh (Cloud Run/Modal).
  • GPU spot/preemptible cho batch không gấp.
  • Chấm tầng: no_ai=true lọc nhanh, chỉ gọi Claude cho bài cần.
  • TOEIC_MODEL=claude-sonnet-4-6 cho throughput; Opus chỉ khi cần chất lượng cao nhất.

🏎️ Tăng throughput

  • Scale theo replica, mỗi replica 1 worker / 1 GPU.
  • WHISPER_DEVICE=cuda nếu có GPU.
  • /grade-batch với max_concurrency=4–8 khi backend cloud.
  • Autoscale theo độ dài queue (Phương án B) chuẩn hơn theo CPU.

⏱️ Tránh timeout

  • Tăng request timeout của nền tảng & LB.
  • Tăng client timeout khi gọi batch.
  • Lớp đông → cắt nhỏ batch (≤ giới hạn 100 file) hoặc chuyển sang job async.

🔐 Secrets đúng cách

# Cloud Run: gắn secret từ Secret Manager
gcloud run deploy speaking-grader \
  --image REGION-docker.pkg.dev/PROJECT/repo/speaking-grader \
  --region asia-southeast1 --port 8000 \
  --memory 4Gi --cpu 2 --timeout 3600 --min-instances 1 \
  --set-secrets ANTHROPIC_API_KEY=anthropic-key:latest \
  --set-env-vars TOEIC_MODEL=claude-sonnet-4-6,WHISPER_MODEL=base
Không bao giờ commit .env thật hay nhúng key vào image. File .env hiện đã nằm trong thư mục project — kiểm tra nó đã được .gitignore bỏ qua trước khi build/push.

6 Bản GPU (khi cần tốc độ)

faster-whisper trên GPU cần CUDA 12 + cuDNN (đã khai báo nvidia-cublas-cu12, nvidia-cudnn-cu12 trong requirements.txt). Dùng base image CUDA và bật cuda:

FROM nvidia/cuda:12.4.1-cudnn-runtime-ubuntu22.04
RUN apt-get update && apt-get install -y --no-install-recommends \
      python3 python3-pip ffmpeg && rm -rf /var/lib/apt/lists/*
WORKDIR /app
COPY requirements.txt .
RUN pip3 install --no-cache-dir -r requirements.txt
ENV WHISPER_MODEL=medium WHISPER_DEVICE=cuda
RUN python3 -c "from faster_whisper import WhisperModel; WhisperModel('medium')"
COPY src ./src
CMD ["sh","-c","uvicorn src.api:app --host 0.0.0.0 --port 8000 --workers 1"]
  • Chạy với docker run --gpus all ... (hoặc nền tảng tự cấp GPU).
  • GPU rẻ & đủ mạnh: T4 / L4; medium/large-v3 cần VRAM lớn hơn.
  • 1 GPU = 1 replica = 1 worker. Cần nhiều thông lượng → thêm replica, không thêm worker.

Bật multi-tier ASR (fast / review) — chỉ trên GPU

Chỉ cài khi thật sự cần lane fast (insanely_fast_whisper) và review (whisperx) riêng. Các gói này kéo theo torch/pyannote rất nặng — không nên đưa vào image CPU.

# Thêm vào Dockerfile bản GPU, SAU khi cài requirements.txt
RUN pip3 install --no-cache-dir whisperx insanely-fast-whisper
  • Cài xong thì bỏ phần override TOEIC_ASR_BACKEND_* trong docker-compose.yml để dùng đúng từng lane.
  • Chưa cài: cứ ép cả 3 tầng về faster_whisper (mục 4) — an toàn cho CPU.

7 CI/CD (GitHub Actions)

# .github/workflows/deploy.yml (rút gọn)
on: { push: { branches: [main] } }
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: pip install -r requirements.txt && pytest -q   # test trước khi deploy
      - run: docker build -t $IMAGE .
      - run: docker push $IMAGE
      - run: gcloud run deploy speaking-grader --image $IMAGE  # hoặc modal deploy / runpod

Pipeline: test → build image → push registry → deploy. Key/secret lấy từ GitHub Secrets, không hard-code.

8 Checklist trước khi lên production

  • ffmpeg có trong image (faster-whisper bắt buộc).
  • Model Whisper được prefetch/cache → cold start chấp nhận được.
  • ANTHROPIC_API_KEY nạp qua Secret Manager, KHÔNG nằm trong image/git.
  • .dockerignore loại .venv .git data outputs logs .env.
  • uvicorn --workers 1 mỗi replica; scale bằng replica (Whisper không đa luồng).
  • Request & LB timeout đủ lớn cho /grade-batch (hoặc đã chuyển job async).
  • Log ra stdout; GET /health gắn vào liveness/readiness probe.
  • Sizing RAM/VRAM khớp WHISPER_MODEL (base CPU ~2–4GB; medium/large GPU nhiều hơn).
  • HTTPS + xác thực (API key/JWT) trước endpoint — tránh lạm dụng tốn tiền Claude.
  • Kết quả lưu vào S3/GCS/DB nếu cần giữ (container ephemeral).
  • Theo dõi chi phí Claude API + cảnh báo ngân sách.
