1

ASR — Speech-to-Text (local)

src/asr.py · transcribe()

Dùng faster-whisper chạy cục bộ để chuyển audio thành transcript kèm mốc thời gian từng từ và độ tự tin. Model được cache trong process; bật vad_filter để lọc khoảng lặng.

in audio_path, model_size, device out Transcription{ text, words[], language, duration }

Mỗi Word có text, start, end, probability — đây là dữ liệu gốc để bước 2 tính tốc độ nói & quãng ngắt.

2

Features — Đặc trưng khách quan (KHÔNG dùng AI)

src/features.py · extract_features()

Tính số liệu định lượng từ word timestamps. Với Read Aloud, so transcript với script tham chiếu để ra WER và chi tiết lỗi.

in Transcription, reference_script? out Features (+ AccuracyMetrics nếu có script)
Nhịp điệuspeech_rate_wpm, pause_count, total/longest_pause_sec (ngưỡng > 0.3s)
Thời lượngspeaking_duration_sec, audio_duration_sec, silence_sec
Từ đệmfiller_count (um, uh, like, you know…)
Tín hiệu phụ ⚠️avg/min_word_probability — KHÔNG phải điểm phát âm
Độ chính xác (Read Aloud)wer, substitutions, insertions, deletions, hits (jiwer, fallback Levenshtein)
3

Gating — Kiểm tra rule-based (rẻ, ổn định, KHÔNG dùng AI)

src/gating.py · evaluate()

Bắt sớm các trường hợp hiển nhiên để khỏi tốn lượt gọi Claude và để ổn định điểm task_completion.

Audio rỗngword_count == 0 → is_empty=True → bỏ qua Claude
Quá ngắnspeaking_duration < 5s hoặc word_count < 3 → floor = very_low
Ngắn hơn kỳ vọngtỉ lệ < 40% expected_duration_sec → floor = low

✖ is_empty / --no-ai

Không gọi Claude. scores = None. Đi thẳng tới bước Report (chỉ transcript + features).

✓ Có lời nói & không --no-ai

Chuyển task_completion_floor + reasons sang bước 4 để Claude không chấm vượt trần.
4

Scoring — Chấm bằng Claude (structured output)

src/scoring.py · score()

Dựng system prompt từ rubric + user prompt chứa JSON (đề bài, script, transcript, số liệu khách quan, cờ gating). Gọi client.messages.parse(...) với output_format=SpeakingResult → Claude trả JSON đúng schema, không cần tự parse.

in Config, QuestionType, prompt, script, Transcription, Features, GatingResult out SpeakingResult
Bằng chứng PRIMARYobjective_metrics (WPM, pauses, fillers, WER…)
Bằng chứng SECONDARYphân tích nội dung transcript
Bằng chứng yếu ⚠️ASR confidence — không dùng làm điểm phát âm
ModelTOEIC_MODEL (mặc định claude-sonnet-4-6; đổi opus-4-8 để benchmark)
SchemaSpeakingResult: task_completion, content_relevance, criteria[], estimated_toeic_score (0-200), summary_feedback

Lỗi gọi API → log & return 1. parsed_output is None (refusal/max_tokens) → raise.

5

Report — Xuất kết quả

src/report.py · build_output() → save_json() → print_report()

Gom tất cả thành 1 dict, lưu JSON đầy đủ (để debug sau) và in ra console (đẹp bằng rich nếu cài, không thì plain text).

file outputs/<audio>__<question>.json console bảng features + điểm từng tiêu chí + điểm TOEIC log logs/app.log
Vì sao tách gating khỏi Claude? Các trường hợp audio rỗng/quá ngắn được bắt bằng luật rẻ & tất định, vừa tiết kiệm chi phí API vừa giữ task_completion ổn định. Claude chỉ chấm phần cần phán đoán ngôn ngữ, với số liệu khách quan làm bằng chứng chính — confidence của Whisper chỉ là tín hiệu phụ.
